# Contributing

If you have cool stuff you want added, feel free to open an issue and I'll try to add it. Otherwise, you can implement it yourself. I apologize for the messy code and appreciate your interest in the project!

### Versioning

Please increment the version number in `scripts/sw.js` and `scripts/script.js` before pushing.
Versioning is as follows: `Major.Update.Patch`. Major updates would include things like complete UI or data processing overhauls that fundamentally change Orpheus. Updates would include things like new features, redoes of a smaller feature, etc. Patches include bug fixes or minor tweaks.

### Bug reporting
Make a Github issue or contact `slothh__` on Discord. The more detailed you can be, the better. Screenshots, logs, data, mappings, layouts, and steps on how to reproduce the issue would be greatly appreciated.

### No AI-generated code
Orpheus is entirely written by humans, and it would be nice to keep it that way.

### Testing
Ideally, all features of Orpheus should be tested before pushing, because a lot of Orpheus is interconnected and changes may be far-reaching. 

If fixing a breaking/critical issue during a competition, less rigorous testing is needed.
